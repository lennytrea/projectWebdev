Multitier Architecture

What is Multitier Architecture?

A way of designing software systems by separating them into distinct layers, each with a specific role.

Key Facts

  • Each tier runs independently and communicates with others.
  • Widely used in web applications and enterprise software.
  • A change in one tier does not require rewriting the others.

The 3 Tiers

  • Presentation Tier - What the user sees and interacts with. Also called the frontend.
  • Logic Tier - The application layer. Also called the backend.
  • Data Tier - The database layer. Stores, retrieves, and manages all data used by the application.

How It Works

  • User interacts with the Presentation Tier
  • The Logic Tier receives the request and processes it .
  • The Data Tier is queried for the relevant data .
  • The result is passed back up and displayed to the user.

Example

  • Presentation - The Gmail interface in browser.
  • Logic - Google's servers processing login and loading emails.
  • Data - The database storing all emails and account information.
